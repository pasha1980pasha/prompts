ESP подключена к Wi-Fi (192.168.1.106, имя max-esp32), IP закреплён в роутере.
Правила работают: при температуре >30.5°C реле выключается (GPIO12,0), при <29.5°C — включается (GPIO12,1).
Всё локально, MQTT/Domoticz/ThingSpeak не используются.

Я использую микроконтроллер ESP32-S3 с прошивкой ESPEasy Mega для локального управления реле по температуре с датчика DS18B20 (интерфейс OneWire, подключён к GPIO1).
Реле управляется через GPIO12:

При температуре ниже 29.5°C — реле включается (GPIO12,1),
При температуре выше 30.5°C — реле выключается (GPIO12,0).
ESP32 подключена к домашней Wi-Fi-сети (TP-LINK_BF1366), имеет статический IP 192.168.1.106, имя хоста — max-esp32.
Всё работает локально: без MQTT, без облаков (ThingSpeak и т.п.), без Domoticz.

У меня ПК с Windows, установлены PowerShell и Python.
Хочу записывать историю температуры на ПК и строить графики за день/неделю/месяц.

Предпочтения:

Минимум зависимостей и сложностей,
Без развёртывания серверов (Mosquitto и т.п.),
Готовые решения: либо встроенный график ESPEasy (60 мин), либо локальный CSV-лог на ПК + график в Excel или Python.
ESPEasy умеет отдавать данные по HTTP: http://192.168.1.106/json?tasknr=1 — там есть значение температуры.

Помоги:

Настроить надёжное логгирование температуры каждые 5 минут на ПК,
Предложить простой способ построения графиков (Excel или Python),
По возможности — автоматизировать запуск логгера при старте ПК.
