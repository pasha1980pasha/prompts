Промт для нового чата (локальный мониторинг ESP32-S3):

Я использую ESP32-S3 с прошивкой ESPEasy Mega для автономного управления реле на GPIO12 по температуре с датчика DS18B20 (подключён к GPIO5).

Логика управления (реализована в Rules):

При температуре ниже 29.5°C → GPIO,12,1 (включить реле)
При температуре выше 30.5°C → GPIO,12,0 (выключить реле)
Сеть:

Устройство подключено к Wi-Fi TP-LINK_BF1366
Статический IP: 192.168.1.106
Имя хоста: max-esp32 или esp32max
Требования:

Вся система — строго локальная: без MQTT, без облаков (ThingSpeak, Domoticz и т.п.), без внешних серверов
Данные должны собираться на моём ПК с Windows
Нужен надёжный способ логгирования температуры каждые 5 минут
Графики должны строиться за день / неделю / месяц — без ограничения 60 минут, как во встроенном графике ESPEasy
Предпочтительные форматы: CSV + Excel или Python с matplotlib
Минимум зависимостей: допустимы только PowerShell или Python (оба установлены)
Желательно автоматическое обновление графика и/или автозапуск логгера при старте Windows
Дополнительно:

ESP32 отдаёт данные в формате JSON: http://192.168.1.106/json
В ответе есть задача с именем Temperature, значение доступно как Sensors[0].TaskValues[0].Value
Управление реле с ПК тоже приветствуется (через HTTP-запросы)
Файловый архив обязателен — данные не должны теряться после перезагрузки ПК
Цель:
Получить единый, простой и стабильный рабочий скрипт (PowerShell или Python), который:

Опрашивает ESP32 каждые 5 минут
Записывает температуру + временную метку в temperature.csv
Позволяет строить графики за любой период
(Опционально) содержит кнопки управления реле
